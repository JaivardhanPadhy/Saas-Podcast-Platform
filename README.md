# Saas-Podcast-Platform

## Introduction
Welcome to the Saas-Podcast-Platform, a unique SaaS application that leverages AI-powered features to enhance podcast creation and management. Built with Next.js 14 and Convex, this platform offers a range of functionalities designed to provide a seamless user experience for both creators and listeners.

## Features
<div>
  
<li>Robust Authentication: Secure and reliable user login and registration system.</li>
<li>Modern Home Page: Showcases trending podcasts with a sticky podcast player for continuous listening.</li>
<li>Discover Podcasts Page: Dedicated page for users to explore new and popular podcasts.</li>
<li>Fully Functional Search: Allows users to find podcasts easily using various search criteria.</li>
<li>Create Podcast Page: Enables podcast creation with text-to-audio conversion, AI image generation, and previews.</li>
<li>Multi Voice AI Functionality: Supports multiple AI-generated voices for dynamic podcast creation.</li>
<li>Profile Page: View all created podcasts with options to delete them.</li>
<li>Podcast Details Page: Displays detailed information about each podcast, including creator details, number of listeners, and transcript.</li>
<li>Podcast Player: Features backward/forward controls, as well as mute/unmute functionality for a seamless listening experience.</li>
<li>Responsive Design: Fully functional and visually appealing across all devices and screen sizes.</li>
</div>

## Installation
To get started with the Saas-Podcast-Platform, follow these steps:

1. **Clone the Repository:**
    ```bash
    git clone https://github.com/JaivardhanPadhy/Saas-Podcast-Platform.git
    cd Saas-Podcast-Platform
   ```


2. **Copy code:**
   ```bash
    git clone https://github.com/JaivardhanPadhy/Saas-Podcast-Platform.git
    cd Saas-Podcast-Platform
   ```

3.**Install dependencies:**

  ```bash
   npm install
```
Set up environment variables:
Create a .env file in the root directory and add the necessary environment variables. Refer to the .env.example file for guidance.

4.**Run the application:**

```bash
npm run dev

```
The application should now be running on <a>http://localhost:3000</a>
